🧰 Guía de insfraestructura del curso

Python – Desarrollo de Software (con apoyo de IA)

🎯 Objetivo del curso

Este curso busca que el alumno pase de resolver algoritmos a desarrollar software real, aprendiendo:

  • a trabajar con proyectos
  • a usar herramientas profesionales
  • a escribir código limpio y probado

Las herramientas están elegidas para reducir fricción, no para agregar complejidad.

Introducción

🧠 Idea clave del curso

El alumno escribe código.
Las herramientas solo preparan el entorno, ordenan el código y verifican que funcione.

🧭 Organización y trabajo profesional

En cursos anteriores resolviste problemas y algoritmos.
En este curso aprenderás a construir software.

Cuando el software crece, no basta con escribir archivos .py:

  • hay múltiples archivos
  • hay librerías externas
  • hay pruebas
  • hay otras personas trabajando en el mismo código

Por eso es necesario organizar el trabajo y seguir una infraestructura común.

Este documento presenta la infraestructura de trabajo que se usará durante todo el curso y explica:

  • qué herramientas se usarán
  • para qué sirve cada una
  • qué conceptos necesitas entender

No es para memorizar comandos, sino para entender cómo se trabaja en un proyecto real.

📘 Conceptos básicos que usarás todo el curso

Gestor de paquetes
Herramienta que instala y administra librerías de un proyecto.

Dependencias
Librerías externas que el proyecto necesita para funcionar.

Entorno virtual
Espacio aislado donde viven las dependencias de un solo proyecto.

📌 En este curso, uv se encarga automáticamente de todo esto.

🐍 1. Lenguaje

Instalación de Python

Para el curso se requiere Python 3.11 o superior.

🧭 Paso 1: Descargar Python

Ir al sitio oficial:

👉 https://www.python.org

  1. Haz clic en el botón grande que dice Download Python
  2. Se descargará un archivo instalador en tu computadora

💻 Paso 2: Instalar Python según tu sistema

🪟 Windows

  1. Abre el archivo descargado (doble clic)

  2. ⚠️ IMPORTANTE: Antes de instalar, activa esta casilla:

    ✅ Add Python to PATH

  3. Haz clic en:

    👉 Install Now

  4. Espera a que termine la instalación

🍎 macOS

  1. Abre el archivo .pkg descargado
  2. Sigue los pasos del instalador (Next → Next → Install)
  3. No necesitas cambiar opciones

🐧 Linux (Ubuntu / Debian)

Abre una terminal y ejecuta:

sudo apt update
sudo apt install python3 python3-pip

🧪 Paso 3: Verificar instalación

Ahora vamos a comprobar que Python quedó bien instalado.

📌 Abrir terminal

  • Windows → buscar "cmd" o "PowerShell"
  • macOS → abrir Terminal
  • Linux → abrir Terminal

Ejecutar:

python --version

o si no funciona:

python3 --version

✅ Resultado esperado

Debe aparecer algo como:

Python 3.11.5

🚨 Problemas comunes

❌ "python no se reconoce como comando" (Windows)

Significa que no se agregó al PATH.

Solución:

  • Reinstalar Python

  • Asegurarte de marcar:

    ✅ Add Python to PATH

❌ Se abre Microsoft Store en Windows

Es normal si no está bien configurado Python.

Solución:

  • Reinstalar desde python.org
  • Activar "Add to PATH"

❌ Python versión menor a 3.11

Descargar la versión correcta desde:

👉 https://www.python.org/downloads/

⚡ 2. Gestor del proyecto: uv

2.1 ¿Qué es uv?

uv es la herramienta que gestiona el proyecto Python. Se encarga de:

  • crear el proyecto
  • crear y administrar el entorno virtual
  • instalar dependencias
  • ejecutar Python y herramientas

📌 El alumno no usa pip ni venv directamente.

pip es el gestor de paquetes tradicional de Python. Instala, actualiza y elimina librerias.

.venv es la carpeta que contiene el entorno virtual de un proyecto. Dentro de .venv viven: Python del proyecto, las dependencias instaladas, configuraciones aisladas. Cada proyecto tiene su propio .venv, lo que evita conflictos entre proyectos.

uv crea y gestiona automáticamente la carpeta .venv. El alumno no necesita entrar ni modificarla.

2.2 Instalación de uv

Antes de usar uv, debe estar instalado en la computadora.

  • macOS / Linux:

    curl -LsSf https://astral.sh/uv/install.sh | sh

curl es una herramienta de línea de comandos para descargar contenido desde internet.

  • Windows:

    winget install astral-sh.uv

    winget es el gestor de paquetes de Windows.

    Cómo verificar si winget está instalado En la terminal (PowerShell o CMD):

    winget --version

Verificación de instalación de uv

Cuando se instala un programa, se pueden añadir rutas al PATH.

Las terminales abiertas no se enteran de esos cambios. Asi que Cierra completamente PowerShell / CMD / Windows Terminal. Ábrela de nuevo y ejecuta:

uv --version

debería aparecer un número que es la versión de uv.

2.3 Crear un proyecto

uv init nombre-proyecto

Esto crea:

  • pyproject.toml
  • README.md
  • el entorno del proyecto

🌱 Relación con Git

  • Si Git está instalado, uv init crea el repositorio automáticamente y será controlado por git.
  • Si Git no está instalado, el proyecto funciona igual.

Cuando Git se vea en clases posteriores, podrás hacer:

git init
git add .
git commit -m "Proyecto inicial"

2.4 Instalar herramientas y dependencias del proyecto

cd nombre-proyecto
uv add pytest black isort --dev

¿Qué hace este comando?

  • uv add → agrega una dependencia al proyecto
  • pytest → herramienta para ejecutar pruebas
  • black → formateador automático de código
  • isort → ordena los imports
  • --dev → indica que son dependencias de desarrollo

📌 Las dependencias de desarrollo:

  • se usan para programar y probar
  • no son necesarias cuando el programa se ejecuta en producción

nota

Si el programa no funciona sin esa librería → es de producción.
Si solo ayuda a desarrollar, probar o dar formato→ es de desarrollo.

Todas estas dependencias quedan registradas en pyproject.toml.

2.5 Ejecutar código y herramientas

uv run python archivo.py
uv run pytest

No es necesario activar el entorno virtual.

2.6 Mantener y actualizar dependencias

  • Sincronizar el proyecto (por ejemplo, al clonar un repositorio):

    uv sync

  • Actualizar dependencias:

    uv sync --upgrade

🖥️ 3. Editor de código

VS Code es el editor que usarás para escribir y ejecutar tu código.

Paso 1: Descargar VS Code

Ir al sitio oficial:

👉 https://code.visualstudio.com

Haz clic en Download según tu sistema operativo.

Paso 2: Instalar VS Code

🪟 Windows

  1. Abre el archivo descargado
  2. Acepta términos
  3. Marca estas opciones (importante):
    • ✅ Add to PATH
    • ✅ Add "Open with Code"
  4. Install

🍎 macOS

  1. Abrir el archivo .zip
  2. Arrastrar Visual Studio Code a Applications

🐧 Linux (Ubuntu)

sudo snap install code --classic

Paso 3: Abrir VS Code

  • Ejecuta VS Code
  • Verás una ventana inicial

Paso 4: Instalar extensiones necesarias

En VS Code:

  1. Ir a Extensions (ícono de cuadritos)

  2. Buscar e instalar:

  • Python (Microsoft)
    Permite que VS Code entienda el lenguaje Python.

  • Pylance
    Da autocompletado, detección de errores y análisis de tipos.

  • GitHub Copilot
    Asistente de IA que sugiere código y tests.

  • Black Formatter
    Aplica el formato de código automáticamente desde el editor.

  • isort
    Ordena los imports del código.

  • Python Test Explorer
    Permite ejecutar y ver tests pytest desde VS Code.

📌 Las extensiones no instalan herramientas.
Solo utilizan las que uv instaló en el proyecto.

🌱 4. Uso de Git y GitHub desde VS Code (flujo básico)

Esta sección explica cómo trabajar sin usar la terminal, directamente desde VS Code.

1. Abrir tu proyecto en VS Code

  • Abrir VS Code
  • Ir a File → Open Folder
  • Seleccionar la carpeta de tu proyecto

2. Inicializar Git (si aún no está)

En la barra lateral izquierda:

  1. Haz clic en el ícono de Source Control (ramita)
  2. Si aparece botón Initialize Repository, haz clic ahí

3. Guardar cambios (commit)

  1. Modifica o crea archivos
  2. Ve a Source Control
  3. Verás los archivos modificados
  4. Escribe un mensaje (ej: primer commit)
  5. Haz clic en Commit

📌 Esto guarda una versión del proyecto

4. Publicar en GitHub

Después del primer commit:

  1. Haz clic en Publish to GitHub
  2. Elegir:
    • repositorio público o privado
  3. Confirmar

📌 VS Code crea y conecta automáticamente el repositorio

5. Flujo de trabajo diario

Cada vez que cambies algo:

  1. Guardar archivos
  2. Ir a Source Control
  3. Escribir mensaje
  4. Click en Commit
  5. Click en Sync Changes

6. Ver tu código en GitHub

  1. Ve a https://github.com
  2. Entra a tu repositorio
  3. Verás:
    • tus archivos
    • historial de commits

🚨 Problemas comunes

❌ No aparece "Publish to GitHub"

→ No has iniciado sesión en GitHub desde VS Code

Solución:
- Ir a Accounts → Sign in with GitHub

❌ Cambios no aparecen en GitHub

→ Falta hacer Sync / Push

🧠 Idea clave

VS Code te permite usar Git sin terminal, pero el concepto sigue siendo el mismo: guardar cambios (commit) y subirlos (push).

📁 6. Estructura estándar de proyecto

Un proyecto Python es una carpeta que contiene: - código - dependencias - configuración

El archivo que define el proyecto es pyproject.toml.

project/
├── src/
│   └── app/
│       ├── __init__.py
│       └── main.py
├── tests/
│   └── test_main.py
├── pyproject.toml
└── README.md

Esta estructura se mantiene durante todo el curso.

🧠 7. Reglas claras del curso

  • Todo lo relacionado con Python → uv
  • Todo lo relacionado con edición → VS Code
  • Código sin formato → Black lo corrige
  • Imports desordenados → isort los ordena
  • Código sin tests → no está terminado
  • Copilot → ayuda, no reemplaza

🏁 Frase resumen

uv gestiona el proyecto, VS Code edita, pytest prueba,
black ordena, isort limpia y Copilot apoya.

Notas Adicionales

¿Qué es un framework?

Un framework es un conjunto de reglas, estructuras y herramientas predefinidas que sirven como base para construir un tipo específico de aplicación.

El framework:

  • define cómo se organiza el proyecto
  • controla parte del flujo del programa
  • ofrece componentes listos para usar

En lugar de que el programa llame a tu código, el framework llama a tu código (inversión de control).

Frameworks clásicos en Python

  • Django → aplicaciones web completas (sitios web, sistemas administrativos)
  • Flask → aplicaciones web pequeñas y medianas
  • FastAPI → APIs y servicios web

Estos frameworks se usan mucho en la industria.

¿Por qué no los usamos en el curso?

Porque los frameworks:

  • imponen una estructura desde el primer día
  • esconden parte de la lógica detrás de convenciones
  • introducen muchos conceptos al mismo tiempo

En este curso primero aprenderás a:

  • organizar un proyecto
  • escribir código claro
  • probar tu código

Una vez dominada la base, aprender frameworks será mucho más sencillo.
