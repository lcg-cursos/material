Organizar el proyecto antes de escribir código

Hasta ahora ya hicimos varias cosas importantes:

  1. Entendimos el problema biológico.
  2. Definimos los requisitos del programa.
  3. Analizamos casos de prueba.

Ahora vamos a hacer algo que es muy común en proyectos reales de software:
organizar el proyecto y documentar el diseño antes de escribir código.

Esto nos ayuda a:

  • mantener el proyecto ordenado
  • guardar el contexto del problema
  • documentar cómo pensamos resolverlo
  • facilitar el uso de herramientas como GitHub Copilot en el futuro

En esta clase vamos a usar:

  • uv para manejar el proyecto de Python
  • git para control de versiones
  • GitHub para alojar el repositorio
  • VS Code como entorno de desarrollo

1. Crear el directorio del proyecto

En la terminal:

mkdir regulatory-network
cd regulatory-network

2. Inicializar el proyecto de Python con uv

uv init regulatory-network

Esto crea la base del proyecto de Python.

Estructura inicial aproximada:

regulatory-network/
│
├── pyproject.toml
└── README.md

3. Crear directorios del proyecto

Ahora creamos carpetas para organizar código y datos.

mkdir src
mkdir data
mkdir docs

La estructura queda así:

regulatory-network/
│
├── src/
├── data/
├── docs/
│
├── pyproject.toml
└── README.md

4. Crear los documentos del proyecto

Vamos a crear tres archivos importantes:

  • context.md → describe el problema
  • test_cases.md → contiene los casos de prueba
  • design.md → describe el algoritmo y el diseño de la solución
touch docs/context.md
touch docs/test_cases.md
touch docs/design.md

Ahora la estructura es:

regulatory-network/
│
├── src/
│
├── data/
│
├── docs/
│   ├── context.md
│   ├── test_cases.md
│   └── design.md
│
├── README.md
└── pyproject.toml

5. ¿Qué va en cada archivo?

context.md

Describe el problema y el contexto de los datos.

Ejemplo:

# Context

Este proyecto analiza una red de regulación genética.

Los datos contienen interacciones entre factores de transcripción (TF) y genes.

Formato de los datos:

TF gene effect

Ejemplo:

AraC araA + AraC araB - LexA recA - ```

Objetivo del programa:

Generar una tabla que indique para cada TF:

  • Nombre del TF (esta solumna debe estar ordenada)
  • total de genes regulados
  • lista de genes regulados (ordenada) ```

test_cases.md

Aquí guardamos los casos de prueba que ya discutimos en clase.

Ejemplo:

# Casos de prueba

## Caso 1

Entrada:

AraC araA +
AraC araB -

Salida esperada:

AraC 2 araA, araB

design.md

Aquí vamos a escribir el algoritmo de la solución.

Es importante notar que el algoritmo no es contexto del problema,
sino parte del diseño de la solución.

Ejemplo:

# Diseño de la solución

## Algoritmo versión 1

1. Crear una lista de TFs encontrados.

2. Recorrer todas las interacciones.

3. Para cada interacción:
   - obtener el TF
   - si el TF no está en la lista, agregarlo.

4. Para cada TF encontrado:
   - crear una lista de genes
   - recorrer las interacciones
   - agregar genes asociados

5. Contar genes por TF.

6. Imprimir la tabla resumen.

Más adelante analizaremos este algoritmo para decidir qué estructuras de datos necesitamos.

6. Crear el archivo del programa

Ahora creamos el archivo donde irá el código.

touch src/regulon_summary.py

Estructura final del proyecto:

regulatory-network/
│
├── src/
│   └── regulon_summary.py
│
├── data/
│
├── README.md
├── context.md
├── test_cases.md
├── design.md
│
└── pyproject.toml

7. Inicializar Git

Vamos a empezar a registrar los cambios del proyecto.

Sino esta inicializado git, ejecuta esto:

git init

Agregar todos los archivos:

git add .

Crear el primer commit:

git commit -m "Initial project structure"

O bien conecta desde VSC con git como lo viste en clases anteriores.

8. Abrir el proyecto en VS Code

Desde la terminal:

code .

Ahora podrás ver todo el proyecto en el explorador de archivos de VS Code.

9. Subir el proyecto a GitHub

a) Via command line

Primero crea un repositorio en GitHub llamado:

regulatory-network

Luego conecta tu repositorio local:

git remote add origin https://github.com/USUARIO/regulatory-network.git
git branch -M main
git push -u origin main

b) desde VSC

Crea el repositorio desde VSC despues de hacer commit.

Importante Recuerda las buenas practicas al nombrar los commits.

Resultado

Tu proyecto ahora contiene:

  • contexto del problema (context.md)
  • casos de prueba (test_cases.md)
  • diseño de la solución (design.md)
  • código del programa (src/regulon_summary.py)
  • control de versiones con git
  • repositorio remoto en GitHub

Este flujo refleja cómo se trabaja en proyectos reales:

problema
↓
requisitos
↓
casos de prueba
↓
diseño del algoritmo
↓
estructuras de datos
↓
implementación
