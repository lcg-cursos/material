Introducción

En la lección anterior aprendimos a trabajar con variables, tipos básicos y cadenas de texto en Python. También escribimos pequeños programas para manipular secuencias de DNA utilizando funciones ya existentes del lenguaje.

En esta lección comenzaremos a construir programas más generales utilizando control de flujo. El control de flujo permite decidir qué instrucciones se ejecutan y cuántas veces se ejecutan dentro de un programa.

Los dos mecanismos principales que estudiaremos son:

  • Condicionales → permiten tomar decisiones.
  • Ciclos (loops) → permiten repetir operaciones.

Además reforzaremos tres prácticas importantes para la programación científica:

  1. Diseñar primero el algoritmo antes de escribir código.
  2. Realizar trazado manual para entender cómo evoluciona un programa.
  3. Pensar en casos límite para validar nuestras soluciones.

Como ejemplo utilizaremos secuencias de DNA y aprenderemos a generar k-mers, subsecuencias de longitud k que se utilizan ampliamente en bioinformática.

Idea clave (por qué hoy NO basta con “buscar la función”): En Python existen funciones útiles (como count()), pero no siempre existe una función lista para resolver el problema que queremos. Por eso hoy aprenderemos a construir soluciones con algoritmos + loops.

1. Valores booleanos (True y False)

Antes de entender los condicionales es importante comprender los valores booleanos.

En Python un valor booleano puede ser:

  • True (verdadero)
  • False (falso)

Estos valores aparecen cuando evaluamos expresiones lógicas.

Ejemplo:

5 > 3

Resultado:

True

Conversión a booleano

Python puede convertir muchos tipos de datos a valores booleanos utilizando la función bool().

bool(0)
bool(1)
bool("")
bool("ATG")

Resultado:

False
True
False
True

Reglas generales:

Reglas generales (valores considerados False): - 0, 0.0 - "" (cadena vacía) - [] (lista vacía) - None

La mayoría de los otros valores se consideran True. Por ejemplo:

bool(2)
bool(-1)
bool("atg")
bool([27, "octubre", 1997])

2. Condicionales (if)

Los condicionales permiten que un programa tome decisiones dependiendo del resultado de una condición lógica.

Un programa puede ejecutar diferentes bloques de código dependiendo de si una condición es verdadera (True) o falsa (False).

Sintaxis básica

if condicion:
    instrucciones

Si la condición es verdadera, se ejecutan las instrucciones dentro del bloque.

Uso de else

Podemos definir qué ocurre cuando la condición no se cumple.

if condicion:
    instrucciones
else:
    otras_instrucciones

Ejemplo:

nt = "A"

if nt in "GC":
    print("Nucleótido GC")
else:
    print("No es GC")

Uso de elif

Cuando existen varias condiciones posibles, usamos elif (else-if).

if condicion1:
    instrucciones
elif condicion2:
    instrucciones
else:
    instrucciones

Ejemplo:

nt = "A"

if nt == "A":
    print("Adenina")
elif nt == "T":
    print("Timina")
elif nt == "G":
    print("Guanina")
elif nt == "C":
    print("Citosina")
else:
    print("Carácter desconocido")

Operadores de comparación

Las condiciones suelen utilizar operadores de comparación:

Operador Significado
== igual a
!= diferente
> mayor que
< menor que
>= mayor o igual
<= menor o igual

Ejemplo:

x = 5

if x > 3:
    print("x es mayor que 3")

Operadores lógicos

Permiten combinar múltiples condiciones.

Operador Significado
and ambas condiciones deben ser verdaderas
or al menos una condición debe ser verdadera
not invierte la condición

Ejemplo:

nt = "G"

if nt == "G" or nt == "C":
    print("GC nucleótido")

Otro ejemplo:

length = 10

if length > 0 and length < 100:
    print("longitud razonable")

Verdad implícita en Python (truthiness)

En Python muchas estructuras pueden evaluarse directamente en un if sin escribir una comparación explícita.

A esto le llamamos truthiness: algunos valores se evalúan automáticamente como True o False en contextos booleanos (como if).

Ejemplo:

seq = ""

if seq:
    print("Hay secuencia")
else:
    print("Secuencia vacía")

recomendaciones PEP8 (mínimas para esta clase)

1) Espacios alrededor de operadores - ✅ if x > 5: - ❌ if x>5:

2) No comparar con True/False - ✅ if flag: - ✅ if not flag: - ❌ if flag == True: - ❌ if flag == False:

3) Usar if seq: para verificar “tiene contenido” - ✅ if seq: - ❌ if len(seq) > 0:

Nota: esto aprovecha el truthiness de Python.

Extra (estilo más claro): - ✅ if nt in "GC": - ❌ if nt == "G" or nt == "C":

3. Ciclos for

En Python los ciclos for se utilizan para recorrer objetos iterables.

Objetos iterables

Un iterable es cualquier objeto que puede devolver sus elementos uno por uno.

Ejemplos de iterables:

  • cadenas de texto
  • listas
  • tuplas
  • diccionarios
  • archivos

Esto significa que podemos recorrerlos con un ciclo for.

Ejemplo:

for nt in "ATGC":
    print(nt)

Salida:

A
T
G
C

Sintaxis

for elemento in secuencia:
    instrucciones

El ciclo toma cada elemento de la secuencia y ejecuta el bloque de código.

Ejemplo 1 — recorrer una secuencia

seq = "ATGC"

for nt in seq:
    print(nt)

Salida:

A
T
G
C

Ejemplo 2 — usar un condicional dentro del for

seq = "ATGCG"

for nt in seq:
    if nt in "GC":
        print(nt, "es GC")

Salida:

G es GC
C es GC
G es GC

Aquí el ciclo recorre cada nucleótido y el condicional permite filtrar solo los nucleótidos de interés.

Función range()()`

La función range() genera una secuencia de números enteros.

Se utiliza frecuentemente cuando queremos recorrer posiciones dentro de una secuencia.

Sintaxis

range(inicio, fin)

También puede usarse con un solo argumento:

range(fin)

Ejemplo

for i in range(5):
    print(i)

Salida:

0
1
2
3
4

Ejemplo 3 — recorrer posiciones con range()

A veces necesitamos conocer la posición dentro de la secuencia.

seq = "ATGC"

for i in range(len(seq)):
    print(i, seq[i])

Salida:

0 A
1 T
2 G
3 C

Esto es útil cuando necesitamos saber dónde ocurre algo dentro de la secuencia.

Variables dentro de for

for i in range(3):
    pass

print(i)

Resultado:

2
  • pass no hace nada, pero permite que el bloque exista.
  • En Python, la variable i sigue existiendo después del ciclo.

Ejemplo 4 — ciclo for anidado

Un for anidado es un ciclo dentro de otro ciclo.

Esto ocurre cuando necesitamos recorrer dos conjuntos de datos.

Ejemplo: comparar cada nucleótido con una lista de nucleótidos válidos.

seq = "ATG"
valid = "ATGC"

for nt in seq:
    for v in valid:
        if nt == v:
            print(nt, "es válido")

En este caso:

  • el ciclo externo recorre la secuencia
  • el ciclo interno recorre los nucleótidos válidos

Los ciclos anidados aparecen en muchos algoritmos de bioinformática, por ejemplo:

  • comparación de secuencias
  • búsqueda de patrones
  • cálculo de matrices de similitud

4. Slicing de cadenas

Python permite extraer subsecuencias de una cadena usando slicing.

Sintaxis

cadena[inicio:fin]

El carácter en inicio se incluye.

El carácter en fin no se incluye (límite superior exclusivo).

Ejemplo

seq = "ATGCA"

print(seq[0:3])

Salida:

ATG

Otro ejemplo:

print(seq[1:4])

Salida:

TGC

Slicing fuera del rango

seq = "ATGC"

print(seq[0:100])

Resultado:

ATGC

Python no genera error cuando el slicing excede el tamaño de la cadena; simplemente devuelve lo que existe.

Función input()

La función input() permite que un programa reciba información del usuario mientras se ejecuta.

Esto es útil cuando queremos que el programa trabaje con datos que el usuario introduce desde el teclado.

Sintaxis

input("mensaje")

El texto dentro de los paréntesis se muestra como mensaje para el usuario.

El valor introducido se puede guardar en una variable.

Ejemplo

seq = input("Introduce una secuencia de DNA: ")

print(seq)

Si el usuario escribe:

ATGCGT

La salida será:

ATGCGT

Importante

La función input() siempre devuelve un string.

Ejemplo:

x = input("Introduce un número: ")

print(type(x))

Salida:

<class 'str'>

Por esta razón, cuando queremos trabajar con números debemos convertir el valor.

Convertir el valor a entero

k = int(input("Introduce el valor de k: "))

Ahora k es un entero y podemos usarlo en cálculos.

Ejemplo completo

seq = input("DNA: ").upper()
k = int(input("k: "))

print("Secuencia:", seq)
print("Valor de k:", k)

Tip

Podemos aplicar métodos directamente al resultado de input().

Ejemplo:

seq = input("DNA: ").upper()

Esto convierte automáticamente la secuencia a mayúsculas.

5. Pensamiento Algorítmico

Antes de escribir código debemos pensar cómo resolver el problema paso a paso.

Esto se llama diseñar un algoritmo.

Algoritmo — Generación de k-mers

Entrada:
- Una secuencia de DNA
- Un número entero k

Salida:
- Todas las subsecuencias de longitud k (k-mers)

Pasos

  1. Leer la secuencia.
  2. Leer k.
  3. Recorrer posiciones desde 0 hasta len(seq) - k.
  4. En cada posición, extraer seq[i:i+k].
  5. Imprimir el k-mer.

Trazado Manual / Casos de prueba

El trazado manual nos permite entender cómo evoluciona el algoritmo.

Entrada:

  • seq = ATGCA
  • k = 3
posición k-mer
0 ATG
1 TGC
2 GCA

Esto nos ayuda a verificar que el algoritmo produce los resultados correctos.

Casos Límite

En esta clase no corregimos con excepciones todavía, pero sí debemos probar y describir qué pasa.

  • seq = "" (vacía)
  • k = 1
  • k = len(seq)
  • k > len(seq)
  • k <= 0

Pensar en estos casos permite construir programas más robustos.

6. Buenas prácticas para escribir commits

Los mensajes de commit deben ser claros, breves y describir el cambio realizado.

Una convención ampliamente utilizada es Conventional Commits.

La estructura general es:

tipo: descripción corta

Ejemplo:

feat: generate k-mers

¿Por qué se usa feat?

feat viene de feature, es decir, funcionalidad nueva.

Se usa cuando el commit agrega una capacidad nueva al programa.

Por eso en esta actividad usamos:

  • feat: generate k-mers
  • feat: compute best GC k-mer
  • feat: sliding window visualization

En los tres casos, el programa gana una funcionalidad nueva.

Tipos de commits comunes

feat

Se usa cuando se añade una nueva funcionalidad al programa.

Ejemplo:

feat: compute best GC k-mer

fix

Se usa cuando se corrige un error.

Ejemplo:

fix: correct GC counting bug

docs

Se usa para cambios en documentación.

Ejemplo:

docs: update README instructions

refactor

Se usa cuando se reorganiza o mejora el código sin cambiar lo que hace.

Ejemplo:

refactor: simplify GC calculation

init

Se puede usar para el primer commit del proyecto, cuando se crea la estructura inicial.

Ejemplo:

init: project structure

Reglas simples para buenos commits

Un buen commit debe:

  • describir qué cambio se hizo
  • ser breve y específico
  • usar una descripción clara
  • enfocarse en una modificación concreta

Ejemplos buenos:

feat: generate k-mers
feat: compute best GC k-mer
feat: sliding window visualization

Ejemplos malos:

changes
update
new code

Estos no explican qué cambió.

10. Tips útiles y observaciones interesantes

Estos pequeños ejemplos suelen generar buenas discusiones porque el resultado no siempre es el que los estudiantes esperan.

Tip 1 - Repetir strings

En Python podemos multiplicar cadenas:

print("AT" * 3)

Salida:

ATATAT

Esto es útil para crear separadores o patrones de manera rápida.

Tip 2 — in funciona con muchas estructuras

El operador in no solo funciona con cadenas. También funciona con listas, conjuntos y diccionarios. Esto lo convierte en una herramienta muy útil para verificar pertenencia.

nt = "G"

if nt in "GC":
    print("Es GC")

Más adelante veremos que también funciona con listas:

bases = ["A","T","G","C"]

if nt in bases:
    print("Base válida")

Tip 3 — Las funciones de Python suelen estar optimizadas

En la lección anterior usamos:

seq.count("G")

Aunque hoy aprendimos a resolver problemas con loops, muchas funciones de Python están implementadas en C, por lo que suelen ser muy rápidas.

Regla práctica:

  • Si existe una función del lenguaje que resuelve el problema → úsala.
  • Si no existe → implementa el algoritmo.

Tip 4 — El slicing es extremadamente poderoso

El slicing permite manipular subsecuencias fácilmente.

seq = "ATGCGAT"

print(seq[2:5])

Salida:

GCG

Muchas operaciones de análisis de secuencias en bioinformática utilizan slicing.

Tip 5 — Ventanas deslizantes aparecen en muchos algoritmos

El concepto de ventana deslizante que usamos para generar k-mers es muy común en bioinformática y ciencia de datos.

Se utiliza en:

  • análisis de k-mers
  • búsqueda de motivos
  • regiones ricas en GC
  • análisis de cobertura

Aprender este patrón algorítmico es muy útil porque aparecerá muchas veces más adelante.

Tip 6 — Siempre prueba con ejemplos pequeños

Cuando estés programando, usa secuencias pequeñas que puedas verificar manualmente.

Ejemplo:

seq = ATGCA
k = 3

Esto facilita verificar que el algoritmo funciona correctamente.

Observaciones importantes sobre los ciclos

  • El ciclo for es uno de los mecanismos más utilizados en Python.
  • Permite recorrer cualquier objeto iterable.
  • Es muy común en bioinformática para analizar secuencias.

Ejemplos típicos en bioinformática:

  • recorrer una secuencia DNA
  • comparar dos secuencias
  • analizar subsecuencias (k-mers)
  • contar patrones

Un punto importante es que Python suele favorecer código simple y legible.

Por ejemplo, es más común escribir:

for nt in seq:

que:

for i in range(len(seq)):

La primera versión suele ser más clara y menos propensa a errores.
