Introducción al problema y dinámica de la clase

En esta sesión vamos a trabajar con un problema inspirado en redes de regulación genética. En biología molecular, los factores de transcripción (TF) regulan la expresión de distintos genes, ya sea activándolos (+) o reprimiéndolos (-). A partir de una lista de interacciones regulatorias entre TFs y genes, queremos construir un programa que resuma esa información y nos permita entender mejor cómo se organiza la regulación.

El objetivo inicial será generar una tabla que indique, para cada TF:

  • cuántos genes regula
  • cuáles son esos genes

Más adelante extenderemos el programa para identificar si cada TF actúa principalmente como activador, represor o regulador dual.

Te invito a leer el articulo "How Generative and Agentic AI Shift Concern from Technical Debt to Cognitive Debt" donde Margaret-Anne Storey describe el cognitive debt como la pérdida de comprensión sobre un sistema cuando priorizamos producir código rápido en lugar de entender realmente cómo funciona. Cuando dejamos de pensar y solo ejecutamos soluciones, el sistema se vuelve cada vez más difícil de razonar y mantener.

8. Listas

Qué es una lista

Una lista es una colección ordenada de elementos.

genes = ["araA", "araB", "araC"]

Carateristicas

a. Iterable

Una lista en Python es un objeto iterable. Puedes recorrer sus elementos uno por uno usando un for.

b. Mutabilidad

Las listas son mutables. Eso significa que pueden cambiar después de ser creadas.

Puedes:

  • agregar elementos
  • quitar elementos
  • modificar elementos

Ejemplos:

genes = ["araA", "araB", "araC"]
genes[1] = "araD"
print(genes)
genes.append("araE")
print(genes)

c. heterogéneas Las listas en Python son heterogéneas, lo que significa que pueden almacenar elementos de diferentes tipos de datos dentro de la misma estructura.

d. Ordenadas. Los elementos tienen posición (índice).

Operaciones básicas con listas

Crear

genes = ["araA", "araB", "araC"]

Acceder por índice

print(genes[0])
print(genes[1])

Último elemento

print(genes[-1])

Recorrer

for gene in genes:
    print(gene)

Longitud

print(len(genes))

Verificar pertenencia

print("araA" in genes)
print("lacZ" in genes)

Métodos útiles de listas

append()

Agrega un elemento al final.

genes = ["araA", "araB"]
genes.append("araC")
print(genes)

Salida esperada:

['araA', 'araB', 'araC']

extend()

Agrega varios elementos.

genes = ["araA", "araB"]
genes.extend(["araC", "araD"])
print(genes)

Salida esperada:

['araA', 'araB', 'araC', 'araD']

remove()

Quita un elemento por valor.

genes = ["araA", "araB", "araC"]
genes.remove("araB")
print(genes)

Salida esperada:

['araA', 'araC']

pop()

Quita un elemento por posición y lo devuelve.

genes = ["araA", "araB", "araC"]
ultimo = genes.pop()
print(ultimo)
print(genes)

Salida esperada:

araC
['araA', 'araB']

sort()

Ordena la lista.

genes = ["lacZ", "araA", "recA"]
genes.sort()
print(genes)

Salida esperada:

['araA', 'lacZ', 'recA']

index()

Regresa la posición de un elemento.

genes = ["araA", "araB", "araC"]
print(genes.index("araB"))

Salida esperada:

1

Ejemplos pensados para este problema

genes_arac = []
genes_arac.append("araA")
genes_arac.append("araB")
print(genes_arac)

Salida esperada:

['araA', 'araB']
for gene in genes_arac:
    print(gene)

Salida esperada:

araA
araB

9. Tuplas

Qué es una tupla

Una tupla es una colección ordenada de elementos relacionados.

interaction = ("AraC", "araA", "+")

Aquí los tres valores pertenecen a una misma interacción.

Qué significa que sea inmutable

Las tuplas son inmutables.

Eso significa que no se pueden modificar después de ser creadas.

Ejemplo:

interaction = ("AraC", "araA", "+")

Esto no se puede hacer:

interaction[1] = "araB"

¿Por qué usar una tupla aquí?

Porque una interacción representa un conjunto fijo de datos:

  • TF
  • gene
  • effect

No queremos estar cambiando esos tres campos arbitrariamente una vez definida la interacción.

Operaciones básicas con tuplas

Crear

interaction = ("AraC", "araA", "+")

Acceder por índice

print(interaction[0])
print(interaction[1])
print(interaction[2])

Longitud

print(len(interaction))

Desempaquetar

TF, gene, effect = interaction
print(TF)
print(gene)
print(effect)

Ejemplos útiles

interaction = ("LexA", "recA", "-")
TF, gene, effect = interaction
print(TF, gene, effect)

Salida esperada:

LexA recA -

10. Mutable vs inmutable

Este contraste es importante.

Estructura ¿Mutable? ¿Qué significa?
Lista Sí Se puede modificar después de creada
Tupla No No se puede modificar después de creada
Diccionario Sí Se pueden agregar, cambiar o quitar pares clave-valor

Ejemplo comparativo

Lista mutable

genes = ["araA", "araB"]
genes.append("araC")
print(genes)

Tupla inmutable

interaction = ("AraC", "araA", "+")

No puedes hacer:

interaction[1] = "araB"

Idea didáctica

Para este problema:

  • usamos tuplas para representar una interacción individual
  • usamos listas para guardar muchas interacciones o muchos genes

11. Lista de interacciones

Ahora juntamos ambas ideas.

Una lista puede contener muchas tuplas.

interactions = [
    ("AraC", "araA", "+"),
    ("AraC", "araB", "-"),
    ("LexA", "recA", "-"),
    ("CRP", "lacZ", "+"),
    ("CRP", "lacY", "+")
]

Cómo recorrer esta estructura

for interaction in interactions:
    print(interaction)

Salida esperada:

('AraC', 'araA', '+')
('AraC', 'araB', '-')
('LexA', 'recA', '-')
('CRP', 'lacZ', '+')
('CRP', 'lacY', '+')

Cómo desempaquetar al recorrer

for TF, gene, effect in interactions:
    print(TF, gene, effect)

Salida esperada:

AraC araA +
AraC araB -
LexA recA -
CRP lacZ +
CRP lacY +

Esto es muy útil porque cada vuelta del ciclo te da directamente los tres componentes.

12. Diccionarios

Qué es un diccionario

Un diccionario guarda pares de la forma:

clave -> valor

Ejemplo simple:

ages = {
    "Ana": 20,
    "Luis": 22
}

¿Por qué sirve aquí?

Porque podemos guardar algo como:

TF -> lista de genes

Ejemplo:

regulon = {
    "AraC": ["araA", "araB"],
    "LexA": ["recA"],
    "CRP": ["lacZ", "lacY"]
}

Eso representa exactamente lo que queremos resumir.

Los diccionarios son mutables

Podemos agregar claves nuevas y modificar valores.

regulon = {}
regulon["AraC"] = ["araA", "araB"]
print(regulon)

Salida esperada:

{'AraC': ['araA', 'araB']}

Operaciones básicas con diccionarios

Crear vacío

regulon = {}

Agregar una clave

regulon["AraC"] = ["araA"]

Acceder a un valor

print(regulon["AraC"])

Verificar si una clave existe

print("AraC" in regulon)
print("LexA" in regulon)

Recorrer claves

for TF in regulon:
    print(TF)

Recorrer clave y valor

for TF, genes in regulon.items():
    print(TF, genes)

Métodos útiles de diccionarios

keys()

Devuelve las claves.

print(regulon.keys())

values()

Devuelve los valores.

print(regulon.values())

items()

Devuelve pares clave-valor.

for TF, genes in regulon.items():
    print(TF, genes)

get()

Permite obtener un valor sin provocar error si no existe la clave.

print(regulon.get("AraC"))
print(regulon.get("GadW"))

Ejemplo muy cercano al problema

regulon = {}
regulon["AraC"] = []
regulon["AraC"].append("araA")
regulon["AraC"].append("araB")
print(regulon)

Salida esperada:

{'AraC': ['araA', 'araB']}

13. Implementación parte 1

Ahora sí vamos a implementar la primera versión.

Meta

Construir la tabla:

| TF | Total genes | Lista genes |

Paso 1. Representa los datos

interactions = [
    ("AraC", "araA", "+"),
    ("AraC", "araB", "-"),
    ("LexA", "recA", "-"),
    ("CRP", "lacZ", "+"),
    ("CRP", "lacY", "+")
]

Paso 2. Crea la estructura para agrupar genes por TF

regulon = {}

Paso 3. Recorre las interacciones

for TF, gene, effect in interactions:
    if TF not in regulon:
        regulon[TF] = []
    regulon[TF].append(gene)

Paso 4. Observa la estructura construida

print(regulon)

Salida esperada:

{'AraC': ['araA', 'araB'], 'LexA': ['recA'], 'CRP': ['lacZ', 'lacY']}

Visualización conceptual de la estructura

También puedes imaginarla así:

AraC -> [araA, araB]
LexA -> [recA]
CRP  -> [lacZ, lacY]

Y de forma todavía más gráfica:

regulon
 ├── AraC -> [araA, araB]
 ├── LexA -> [recA]
 └── CRP  -> [lacZ, lacY]

Esta visualización suele ayudar mucho a entender qué se está construyendo en memoria.

Paso 5. Imprime la tabla resumen

Queremos que la salida sea consistente y fácil de leer.

Por eso vamos a ordenar:

  • los TFs (primera columna)
  • la lista de genes de cada TF

Esto también ayuda a que los resultados sean reproducibles cuando probamos el programa.

Paso 5.1 Ordenar los TFs

for TF in sorted(regulon):
    genes = regulon[TF]

La función sorted() devuelve una lista ordenada.

Paso 5.2 Ordenar los genes

genes = sorted(genes)

Paso 5.3 Imprimir la tabla

for TF in sorted(regulon):
    genes = sorted(regulon[TF])
    total = len(genes)
    lista_genes = ", ".join(genes)
    print(TF, total, lista_genes)

Salida esperada:

AraC 2 araA, araB
CRP 2 lacY, lacZ
LexA 1 recA

De esta forma:

  • los TFs aparecerán ordenados alfabéticamente
  • los genes dentro de cada TF también estarán ordenados

Esto produce una salida más clara como:

AraC 2 araA, araB
CRP 2 lacY, lacZ
LexA 1 recA
for TF, genes in regulon.items():
    total = len(genes)
    lista_genes = ", ".join(genes)
    print(TF, total, lista_genes)

Qué aprendiste aquí

  • una lista puede guardar genes
  • una tupla puede representar una interacción
  • un diccionario puede asociar cada TF con su lista de genes
  • append() sirve para acumular
  • items() sirve para recorrer diccionarios

14. Comprensión de listas

La comprensión de listas permite construir listas de forma compacta.

Ejemplo sencillo

nums = [1, 2, 3, 4]
cuadrados = [n * n for n in nums]
print(cuadrados)

Salida esperada:

[1, 4, 9, 16]

Ejemplo con filtro

nums = [1, 2, 3, 4, 5, 6]
pares = [n for n in nums if n % 2 == 0]
print(pares)

Salida esperada:

[2, 4, 6]

Ejemplo aplicado al problema

genes_arac = [gene for TF, gene, effect in interactions if TF == "AraC"]
print(genes_arac)

Salida esperada:

['araA', 'araB']

Cuándo usar comprensión de listas

Úsala cuando quieras construir una lista nueva de forma clara.

No la uses si hace que el código se vuelva difícil de leer.

16. Algoritmo lógico para la extensión

Otra vez: primero la lógica, no la sintaxis.

  1. Para cada TF

  2. iniciar contador de activados en 0

  3. iniciar contador de reprimidos en 0

  4. recorrer todas las interacciones

    si la interacción pertenece a ese TF

    si effect es +

     aumentar activados

    si effect es -

     aumentar reprimidos

  5. decidir el tipo

    si activados > 0 y reprimidos > 0

    tipo = dual

    si activados > 0 y reprimidos = 0

    tipo = activador

    si reprimidos > 0 y activados = 0

    tipo = represor

  6. imprimir resultados

18. Tips prácticos y errores clásicos

Tips con listas

  • Usa append() cuando quieres agregar un solo elemento.
  • Usa extend() cuando quieres agregar varios elementos.
  • Si vas a recorrer una lista y solo leerla, no necesitas modificarla.
  • Si quieres una salida reproducible, recuerda ordenar con sorted() o sort().

Errores clásicos con listas

Error 1: confundir append() con extend()

genes = ["araA", "araB"]
genes.append(["araC", "araD"])
print(genes)

Salida:

['araA', 'araB', ['araC', 'araD']]

Esto agrega una lista dentro de la lista, no dos genes por separado.

Error 2: usar un índice que no existe

genes = ["araA", "araB"]
print(genes[2])

Esto produce error porque solo existen las posiciones 0 y 1.

Tips con tuplas

  • Úsalas cuando los datos de un elemento van juntos.
  • Son ideales para representar registros simples, como una interacción.
  • El desempaquetado hace el código más claro.

Errores clásicos con tuplas

Error 1: intentar modificarlas

interaction = ("AraC", "araA", "+")
interaction[1] = "araB"

Esto provoca error porque la tupla es inmutable.

Error 2: desempaquetar con un número incorrecto de variables

interaction = ("AraC", "araA", "+")
TF, gene = interaction

Esto provoca error porque la tupla tiene tres elementos y solo se dieron dos variables.

Tips con diccionarios

  • Piensa siempre en la forma clave -> valor.
  • En este problema, la clave es el TF y el valor es la lista de genes.
  • Usa if clave not in diccionario antes de crear una entrada nueva.
  • Usa items() para recorrer clave y valor al mismo tiempo.

Errores clásicos con diccionarios

Error 1: acceder a una clave que no existe

regulon = {}
print(regulon["AraC"])

Esto provoca error porque esa clave todavía no fue creada.

Error 2: olvidar inicializar la lista antes de hacer append()

regulon = {}
regulon["AraC"].append("araA")

Esto falla porque primero hay que crear la entrada:

regulon = {}
regulon["AraC"] = []
regulon["AraC"].append("araA")

Error 3: olvidar ordenar la salida

El programa puede funcionar, pero producir una salida difícil de comparar entre pruebas.

Por eso aquí ordenamos:

  • TFs
  • genes por TF
