5. Requisitos

En esta etapa no estamos escribiendo un programa desde cero.
Estamos extendiendo un programa que ya funciona.

Esto implica una buena práctica importante:

No romper lo que ya funciona.

Al agregar nuevas funcionalidades, es recomendable:

  • mantener la lógica existente intacta
  • agregar cambios de forma incremental
  • reutilizar estructuras ya definidas (por ejemplo, interactions)
  • asegurar que la nueva entrada (archivo) produzca el mismo formato que antes
  • validar que los casos de prueba anteriores siguen funcionando

Otra idea clave:

El objetivo no es cambiar el programa, sino cambiar la fuente de los datos.

Antes: python interactions = [(TF, gene, effect), ...]

Ahora:

interactions = leer_archivo(...)

Si logramos eso, el resto del programa no necesita modificarse.

8. Implementación

En esta sección implementamos la nueva funcionalidad del programa: leer el archivo, validar datos, construir interactions y guardar resultados.

🟢 Básico — Lectura y flujo principal

Para resolver el problema necesitamos leer archivos.

leer archivos

Un archivo es una colección de datos almacenados en disco. A diferencia de las variables en Python, los archivos persisten aunque el programa termine.

Leer un archivo implica:

  • abrirlo
  • recorrer su contenido
  • cerrarlo correctamente

open("ruta_del_archivo")

  • se usa para abrir un archivo
  • recibe como argumento la ruta del archivo y el modo.
open("ruta", "modo")

Por ejemplo

open("data/raw/NetworkRegulatorGene.tsv")

Por defecto, abre el archivo en modo lectura ("r").

Para escritura, se usa "w" :

open("data/raw/output.txt", "w")

** with open(...) as f **

  • with es una estructura especial para manejar recursos (como archivos)
  • as f asigna el archivo a una variable (f)

👉 Esto significa:

“abre el archivo y guárdalo en f para poder usarlo”

La estructura with asegura que:

  • el archivo se abre correctamente
  • el archivo se cierra automáticamente al terminar
with open("data/raw/NetworkRegulatorGene.tsv") as f:
    for line in f:
        print(line)

Quitar los saltos de línea

La variable line :

✔ representa una línea completa del archivo.
✔ es un string.
✔ incluye el salto de línea (\n).
✔ normalmente se limpia con .strip(). 

line = "AraC    araA    +
"
print(line.strip())

Elimina espacios y saltos de línea al inicio y final.

Líneas vacías

Los archivos reales pueden contener:

  • líneas en blanco
  • espacios innecesarios
  • saltos de línea adicionales

Si no las ignoramos:

  • split() puede fallar o producir listas vacías
  • podemos introducir datos incorrectos
  • el programa puede comportarse de forma inesperada

Después de limpiar la línea con .strip() podemos:

if not line:
    continue

En Python, una cadena vacía, se evalúa como False (truthiness).

Saltar los comentarios y encabezado

Muchos archivos en bioinformática, traen un encabezado con metadatos, y pueden incluir los nombres de las columnas, por eso es recomendable ignorarlas cuando se quieren hacer los conteos o procesar las lineas de datos.

            # ignorar comentarios
            if line.startswith("#"):
                continue
            
            # ignorar encabezado
            if line.startswith("regulatorId"):
                continue

Separar columnas y validar estructura

              fields = line.split("\t")
            
            # validar número mínimo de columnas
            if len(fields) < 6:
                continue
            
            TF = fields[1]
            gene = fields[3]
            effect = fields[4]

Estos pasos permiten transformar texto en datos estructurados.

Extraer datos y validar contenido (efecto)

Solo aceptaremos interacciones que indique el tipo de efecto.

            # validar effect
            if effect not in ["+", "-"]:

import os

# =========================================
# Lectura del archivo y construcción de interactions
# =========================================

interactions = []

filename = "data/raw/NetworkRegulatorGene.tsv"

if not os.path.exists(filename):
    print("Error: archivo no encontrado")
    exit(1)
else:
    with open(filename) as f:

        for line in f:

            line = line.strip()

            print(f"Leído: {line[:50]}...")

            # Ignorar líneas vacías
            if not line:
                continue

            # Ignorar comentarios
            if line.startswith("#"):
                continue

            # Ignorar encabezado
            if line.startswith("1)regulatorId"):
                continue

            fields = line.split("\t")

            # Validar número mínimo de columnas
            if len(fields) <= 7:
                continue

              # columnas a utilizar
            TF = fields[1]
            gene = fields[3]
            effect = fields[4]

            # Validar effect
            if effect not in ["+", "-"]:
                continue

            interactions.append((TF, gene, effect))

print(interactions)

  • dividir problemas grandes
  • planear antes de programar

👉 Es común en desarrollo real.

¿Qué implica?

Cada TODO representa:

  • una funcionalidad pendiente
  • una decisión por tomar
  • una parte incompleta del programa

👉 Es un puente entre el diseño y el código

¿Cómo se resuelve?

Cada TODO se convierte en código real:

# TODO: leer archivo

→

with open(filename) as f:
    for line in f:
        ...

🤖 Uso con IA

Puedes pedir:

Completa los TODOs en este código siguiendo buenas prácticas

Pero debes:

  • revisar que sea correcto
  • validar que cumpla requisitos
  • ajustar si es necesario

💡 Idea clave

La IA no implementa por ti: te ayuda a avanzar más rápido, pero tú decides.

✅ Buena práctica

  • escribir TODOs claros y específicos
  • eliminarlos una vez implementados
  • usarlos como guía de desarrollo

👉 ayudan a mantener el código organizado y entendible
