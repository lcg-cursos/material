1️⃣ Introducción a argumentos con sys y sys.argv

¿Qué es sys?

sys es un módulo de Python que permite interactuar con el entorno de ejecución.

¿Qué es sys.argv?

Es una lista que contiene los argumentos con los que se ejecuta el programa.

Ejemplo:

python regulon_summary.py input.tsv output.tsv

En Python:

import sys
print(sys.argv)

Salida:

['regulon_summary.py', 'input.tsv', 'output.tsv']

3️⃣ Problemas de sys.argv

Problemas

  • Depende del orden

    Si escribes:

    python regulon_summary.py salida.tsv entrada.tsv

El programa no sabe que el usuario se equivocó; simplemente asigna mal los valores.

  • Todo es string

Todo llega como string. Si algún argumento lo quieres un entero, hay que convertirlo manualmente.

  • Validación manual

Tienes que revisar cantidad de argumentos, formato, mensajes de error, etc. y validarlos para evitar errores.

  • Sin ayuda (--help)

Con sys.argv no tienes automáticamente algo como --help.

  • Poco legible

sys.argv[1], sys.argv[2], sys.argv[3] no es tan legible como nombres de variables.

4️⃣ Introducción a argparse

¿Qué hace argparse?

Libreria que permite definir, validar y documentar argumentos.

Componentes

ArgumentParser()

Crea el objeto que se encargará de definir y procesar la interfaz de línea de comandos

add_argument()

Registra/Define cada argumento que el programa acepta

parse_args()

Lee lo que escribió el usuario en la terminal, valida y construye un objeto con esos valores

Ejemplo

import argparse

parser = argparse.ArgumentParser(description="Procesa regulones")
parser.add_argument("input_file", help="Archivo de entrada")
parser.add_argument("output_file", help="Archivo de salida")

args = parser.parse_args()

print(args.input_file)

ArgumentParser()

parser = argparse.ArgumentParser(description="Procesa regulones")

🔹 ¿Qué hace?

  • Crea el objeto principal que controla los argumentos
  • Define la descripción del programa

🔹 Utilidad

  • Genera automáticamente ayuda con --help
  • Mejora la experiencia del usuario

add_argument()

Define cada argumento que el programa acepta.

Argumentos posicionales

parser.add_argument("input_file", help="Archivo de entrada")
parser.add_argument("output_file", help="Archivo de salida")

🔹 Características:

  • Se pasan sin --
  • Dependen del orden

Argumentos opcionales (flags)

parser.add_argument("--min_genes", type=int, default=0, help="Filtrar TFs")

🔹 Características:

  • No dependen del orden
  • Son más claros

Opciones importantes de add_argument()

help

help="Descripción del argumento"

👉 Aparece en --help

type

type=int

👉 Convierte automáticamente el valor

default

default=0

👉 Define valor por defecto

required

required=True

👉 Obliga a usar el argumento (solo opcionales)

choices

choices=["simple", "full"]

👉 Limita valores posibles

action (flags booleanos)

parser.add_argument("--verbose", action="store_true")

👉 Si aparece → True 👉 Si no → False

3. parse_args()

args = parser.parse_args()

🔹 ¿Qué hace?

  • Lee los argumentos de la terminal
  • Valida automáticamente
  • Aplica tipos
  • Devuelve un objeto
args.input_file
args.output_file
args.min_genes

Ejemplo

import argparse


....

def main():
    
    #  Argumentos del programa
    parser = argparse.ArgumentParser(
        description="Resumen de regulones"
    )
    
    parser.add_argument("input_file", help="Archivo de entrada")
    parser.add_argument("output_file", help="Archivo de salida")

    
    args = parser.parse_args()
